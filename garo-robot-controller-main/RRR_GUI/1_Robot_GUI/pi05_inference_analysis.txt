1. 현재 상황 요약
--------------------------------------------------------------------------------
- 왼팔만 사용하여 "pick up the object" 작업 수행
- 오른팔은 의도적으로 기본자세 고정
- 추론 시 왼팔이 움직이긴 하지만, 카메라 입력과 무관하게
  비슷한 동작만 반복하는 것으로 보임
- 물체 위치를 바꿔도 동일한 궤적을 따라가는 듯한 행동 관찰


2. 수정 완료된 항목들
--------------------------------------------------------------------------------
[O] QUANTILES 정규화 적용 (STATE 입력)
    - stats.json의 min/max 값을 사용하여 정규화
    - 공식: quantile = 2.0 * (linear - min) / (max - min) - 1.0

[O] QUANTILES 역정규화 적용 (ACTION 출력)
    - 공식: linear = (quantile + 1.0) * (max - min) / 2.0 + min
    - position = linear * 2048 + 2048

[O] 그리퍼 반전 설정 (v7 기준)
    - 오른쪽 그리퍼(41): 반전 적용
    - 왼쪽 그리퍼(31): 반전 없음

[O] ACC(가속도) 설정
    - 기본자세 이동 시 ACC=6 적용
    - 추론 중 매 1초마다 ACC=6 재적용

[O] Action Queue 초기화
    - 매 프레임마다 _action_queue.clear() 호출
    - 기존: 50프레임 동안 같은 액션 청크 재사용
    - 수정: 매번 새로운 예측 강제

[O] dtype 일치
    - 이미지, State 모두 bfloat16으로 변환 (모델과 일치)

[O] Serial Timeout 조정
    - 서보 읽기 timeout: 100ms → 20ms
    - 16개 서보 위치 읽기 가능하도록 설정

[O] 오른팔 고정
    - 오른팔 서보(11,12,13,14,15,16,17,41)에는 명령 전송 안 함
    - 왼팔만 모델 출력대로 제어

[O] 언어 명령 일치
    - 학습 데이터: "pick up the object"
    - 추론 기본값: "pick up the object" (동일하게 설정)


3. 의심되는 문제점들
--------------------------------------------------------------------------------

3.1 카메라-관절 데이터 매칭 문제 (주요 의심)

    학습 데이터 수집 시 카메라 이미지와 관절 위치가 제대로 동기화되어
    저장되었는지 의문이 있습니다.

    만약 카메라 프레임과 관절 데이터의 타임스탬프가 어긋났다면:
    - 모델은 "이 이미지일 때 이 관절 위치로 가라"를 잘못 학습
    - 결과적으로 시각 정보와 무관한 평균적인 동작만 학습됨

    또한 3개의 카메라(top, wrist_left, wrist_right)가 학습과 추론에서
    동일한 순서로 매핑되었는지 확인 필요:
    - 학습: camera_keys = ["top", "wrist_right", "wrist_left"]
    - 추론: camera_names = ["top", "wrist_left", "wrist_right"]
    (순서가 다를 수 있음 - 확인 필요)


3.2 시각적 다양성 부족 문제

    96개 에피소드에서 물체 위치가 거의 동일했다면:
    - 모델은 "어떤 이미지가 들어와도 비슷한 궤적" 학습
    - 카메라 입력을 사실상 무시하고 평균 궤적만 출력

    이 경우 카메라가 정상 작동해도 로봇은 항상 같은 동작을 함


3.3 QUANTILES 통계 불일치 가능성

    현재 사용 중인 정규화:
    - min/max 값을 q01/q99로 근사하여 사용

    실제 학습 시 사용된 값:
    - LeRobot 내부에서 계산된 실제 q01, q99 백분위수
    - 이 값들이 stats.json에 저장되지 않았을 수 있음

    만약 실제 q01/q99와 min/max가 다르다면:
    - 입력 정규화가 틀어져서 모델이 이상한 값을 받음
    - 출력 역정규화도 틀어져서 잘못된 위치로 이동


3.4 Task 문자열 단일화 문제

    확인된 사실:
    - 전체 96개 에피소드 모두 task = "pick up the object"
    - 다양한 task 없이 단일 문자열만 사용

    Pi0.5는 VLA(Vision-Language-Action) 모델로서 언어 조건부 학습을
    하도록 설계되었으나, 단일 task로 학습하면:
    - 언어 입력이 사실상 상수가 됨
    - 모델이 언어 조건을 무시하고 학습할 가능성


3.5 추론 속도 불일치

    - 학습 데이터 FPS: 11 Hz
    - 현재 추론 FPS: 약 2-3 Hz (서보 읽기로 인한 지연)

    시간 스케일이 다르면 학습된 동작 타이밍과 맞지 않을 수 있음


4. 학습 데이터 구조 확인 결과
--------------------------------------------------------------------------------

에피소드 데이터 구조:
- episode_data.json: 프레임별 state, action, 이미지 경로
- observation_images_top/: 상단 카메라 이미지
- observation_images_wrist_left/: 왼손목 카메라 이미지
- observation_images_wrist_right/: 오른손목 카메라 이미지

오른팔 데이터 분석:
- 모든 에피소드, 모든 프레임에서 오른팔 state가 동일
- 의도적으로 오른팔 고정 상태로 학습됨 (정상)

왼팔 데이터 분석:
- 에피소드 내에서 왼팔 state가 변화함 (정상)
- 에피소드 간 시작 위치는 유사 (baseline에서 시작)


5. 디버그 로그 분석 결과
--------------------------------------------------------------------------------

최신 로그에서 확인된 사항:
- Cameras: 모두 OK (top, wrist_left, wrist_right 정상 캡처)
- State QUANTILES: 왼팔 값이 프레임마다 변함 (서보 읽기 정상)
- Action QUANTILES: 프레임마다 다른 값 출력 (모델 예측 작동)
- Positions: 왼팔 위치가 변화함 (서보 명령 전송 정상)

그러나:
- Action 출력이 변하더라도 전체적인 궤적 패턴이 유사
- 카메라에 비친 물체 위치와 무관하게 비슷한 동작 반복


6. 권장 조치사항
--------------------------------------------------------------------------------

6.1 즉시 확인 필요:
    [ ] 학습 데이터의 카메라 이미지 직접 확인
        - 에피소드별로 물체 위치가 실제로 달랐는지
        - 카메라 순서(top, wrist_left, wrist_right)가 올바른지

    [ ] 카메라 매핑 순서 확인
        - 학습 시 camera_keys 순서
        - 추론 시 camera_names 순서
        - 불일치 시 수정 필요

6.2 학습 데이터 관련:
    [ ] 새 데이터 수집 시 물체 위치 랜덤화
    [ ] 다양한 task 문자열 사용
    [ ] 카메라-관절 동기화 확인

6.3 정규화 관련:
    [ ] LeRobot 학습 코드에서 실제 q01/q99 계산 방식 확인
    [ ] 필요시 정확한 백분위수 값으로 수정

6.4 대안적 접근:
    [ ] 더 작은 모델(ACT, Diffusion Policy)로 먼저 테스트
    [ ] 동일 데이터로 작은 모델이 작동하면 Pi0.5 설정 문제
    [ ] 작은 모델도 안 되면 데이터 자체 문제


7. 결론
--------------------------------------------------------------------------------

추론 코드는 대부분 올바르게 수정되었습니다. 현재 로그에서:
- 카메라 캡처 정상
- 서보 위치 읽기 정상
- 모델 예측 작동
- 서보 명령 전송 정상

그러나 모델이 카메라 입력에 반응하지 않고 비슷한 동작만 반복하는 것은
학습 단계에서의 문제일 가능성이 높습니다:

1. 카메라-관절 데이터가 제대로 매칭되지 않았거나
2. 학습 데이터의 시각적 다양성이 부족했거나
3. 카메라 순서가 학습과 추론에서 다르거나
4. QUANTILES 정규화 통계가 실제 학습 값과 다르거나

이 중 하나 이상의 문제로 인해 모델이 시각 정보를 제대로 활용하지
못하고 "평균적인 궤적"만 출력하는 것으로 추정됩니다.
