6. 핵심 교훈 정리
================================================================================

6.1 성공 요인 (v02/12K, v43/20K에서 확인)

  [1] Image Augmentation = 필수
      - v43 (Aug ON) → 33% 성공
      - v53 (Aug OFF, 나머지 동일) → 실패
      → 278 에피소드의 작은 데이터에서 augmentation이 결정적

  [2] MEAN_STD 정규화 > QUANTILES (이 데이터셋에서)
      - MEAN_STD 모델(v43/20K, v02/12K)만 추론 성공
      - QUANTILES 모델(v42 등)은 loss는 낮지만 추론 실패
      → Loss scale이 다르므로 낮은 loss ≠ 좋은 성능

  [3] Learning Rate = 2.5e-5가 1.25e-5보다 우수
      - v02 (2.5e-5): 12K에서 loss=0.027 → 최고 성능
      - v01 (1.25e-5): 12K에서 loss=0.047 → 느린 수렴
      → 더 높은 lr이 더 빠르고 깊은 학습을 제공

  [4] Sweet Spot = Warmup 직후 + ~3.5-4.0 epoch
      - v43/20K: epoch ~3.9, loss=0.046 → 성공
      - v02/12K: epoch ~3.6, loss=0.027 → 최고
      - v43/30K: epoch ~5.8, loss=0.033 → 실패 (overfitting)
      → 3~4 epoch가 이 데이터셋의 최적 학습량

  [5] Warmup = 10K가 6K보다 안정적
      - v02 (warmup 10K): loss=0.027 @ 12K
      - v04 (warmup 6K):  loss=0.031 @ 12K
      → 긴 warmup이 더 안정적인 최적점 도달

6.2 실패 요인

  [1] Loss 낮음 ≠ 성능 좋음 (Overfitting)
      - v43: 20K(0.046) > 30K(0.033) > 40K(0.032)
        성공률: 33%  >  10%    >  낮음
      → 278 에피소드로 장시간 학습하면 과적합

  [2] 베이스라인 불일치 (모든 모델 공통)
      - 학습 시 텔레오퍼레이션 시작 자세 ≠ 추론 시 베이스라인
      - 특히 J21(어깨): 차이 1917틱 (~84.5°)
      - J31(그리퍼): 학습 시 반열림(1763), 추론 시 닫힘(0)
      → 이 불일치에도 불구하고 v02/12K는 작동한 것은 놀라운 일반화

  [3] 데이터셋 차이가 성능 좌우
      - v41/v51 (dataset_v41): 실패
      - v43/v02 (dataset_v43): 성공
      → 동일 원본이라도 데이터 전처리 방식이 결정적

  - LR = 2.5e-5가 1.25e-5보다 우수
  - Warmup 10K > 6K (안정성)
  - Loss 0.025~0.030 범위가 최적 (MEAN_STD 기준)
  - Loss가 더 낮아지면(< 0.025) overfitting 위험

【가장 중요한 교훈】
  1. Loss ≠ 성능: 낮은 loss가 좋은 성능을 보장하지 않음
  2. ImgAug = 필수: 작은 데이터셋에서 이미지 증강이 결정적
  3. Sweet Spot 존재: ~3.5 epoch, warmup 직후가 최적
  4. 베이스라인 일치: 텔레오퍼레이션 시작 자세 = 추론 시작 자세
